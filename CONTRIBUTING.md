# Contributing

First off, thank you for considering contributing! It's people like you that make the open source community such a great place to learn, inspire, and create.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork locally: `git clone https://github.com/your-username/dotnet-cqrs-eventsourcing.git`
3. Create your feature branch: `git checkout -b feature/amazing-feature`
4. Run the tests to ensure everything is working correctly before and after your changes.
5. Commit your changes: `git commit -m 'Add some amazing feature'`
6. Push to the branch: `git push origin feature/amazing-feature`
7. Submit a Pull Request.

## Development Requirements

- [.NET 10.0 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)

## Code Style

- Follow the existing coding conventions and style used throughout the project.
- Provide XML documentation comments for all public APIs.
- Keep any existing author headers intact within the files. Do not remove them.

## Issues and Bug Reports

We use GitHub Issues to track public bugs and feature requests.
When opening an issue, please ensure you provide clear and concise reproduction steps, along with expected and actual behavior.
