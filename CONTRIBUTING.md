# Contributing

Thank you for contributing! All contributions — bug reports, feature suggestions, documentation, or code — are welcome.

## How to Contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/my-feature`.
3. Make your changes and ensure tests pass: `dotnet test`.
4. Push and open a Pull Request against `main`.

## Guidelines

- **Keep PRs focused:** One logical change per PR.
- **Documentation:** Update examples if the public API changes.
- **Quality:** Maintain or improve test coverage.
- **Style:** Follow `.editorconfig` (4 spaces, no tabs).
- **XML Docs:** Provide XML documentation comments for all public types and members.

## Issues

Use GitHub Issues to report bugs or request features. Include a minimal reproduction case and your .NET version.
